# experimental-nuxt-app

Тестовый проект на **Nuxt 4** для изучения структуры приложения, локальной разработки, сборки и развёртывания.

Репозиторий: `ssh://git@devops.dev.vbtec.ru:2222/students/experimental-nuxt-app.git`

## О проекте

Проект предназначен для экспериментов с возможностями Nuxt 4:

- разработка интерфейсов на Vue 3;
- работа с файловой маршрутизацией;
- использование серверных обработчиков через Nitro;
- проверка сборки и запуска приложения в production-режиме.

Nuxt 4 поддерживает режимы SSR, prerender/static и deployment на Node.js, а production-сборка формируется в каталоге `.output`. :contentReference[oaicite:0]{index=0}

## Технологии

- **Nuxt 4**
- **Vue 3**
- **Vite** — используется по умолчанию как builder в Nuxt 4 :contentReference[oaicite:1]{index=1}
- **Nitro** — серверная часть Nuxt
- **Node.js** — для локального запуска и production-развёртывания

## Требования

Перед началом работы убедитесь, что установлены:

- **Node.js** актуальной LTS-версии
- **npm** / **pnpm** / **yarn** / **bun**

Nuxt 4 официально поддерживает стандартный сценарий запуска через `dev`, `build` и `preview`, а также рекомендует использовать менеджер пакетов, указанный в проекте. :contentReference[oaicite:2]{index=2}

## Установка

Клонирование репозитория:

```bash
git clone ssh://git@devops.dev.vbtec.ru:2222/students/experimental-nuxt-app.git
cd experimental-nuxt-app
```

Установка зависимостей:

    npm install

или

    pnpm install

или

    yarn install

### Запуск в режиме разработки
    
    npm run dev

После запуска приложение обычно доступно локально по адресу:

    http://localhost:3000

Команда nuxt dev используется для локальной разработки, а порт при необходимости может быть переопределён.

### Сборка проекта

Production-сборка:

    npm run build

После выполнения сборки Nuxt формирует production-артефакты в каталоге `.output.`

Локальный запуск production-сборки
    npm run preview

Команда preview запускает локальный сервер для проверки собранного приложения после `build`. В Nuxt 4 команда `start` является алиасом для preview.

### Возможная структура проекта

Пример типовой структуры Nuxt 4 проекта:

    .
    ├─ app/
    ├─ assets/
    ├─ components/
    ├─ composables/
    ├─ layouts/
    ├─ middleware/
    ├─ pages/
    ├─ plugins/
    ├─ public/
    ├─ server/
    ├─ nuxt.config.ts
    ├─ package.json
    └─ README.md

В Nuxt 4 также используется генерация служебных файлов в директории `.nuxt`, включая конфигурации TypeScript для разных контекстов приложения.

### Переменные окружения

Если проект использует переменные окружения, их следует хранить в локальных env-файлах или задавать через CI/CD.

Пример:

    NUXT_PUBLIC_APP_NAME=Experimental Nuxt App
    NUXT_PUBLIC_API_BASE=/api

Для публичных переменных в Nuxt используется префикс `NUXT_PUBLIC_`.

### Полезные команды

Запуск dev-сервера:

    npm run dev

Сборка:

    npm run build

Предпросмотр production-сборки:

    npm run preview

Вывод информации о текущем Nuxt-окружении:

    npx nuxt info

Команда nuxt info выводит сведения о текущем проекте и окружении.

### Развёртывание

Проект можно развёртывать разными способами:

- как Node.js приложение;
- как статически сгенерированный сайт;
- на платформы, поддерживающие Nuxt/Nitro deployment.

Подход зависит от конфигурации проекта и выбранного режима рендеринга.
